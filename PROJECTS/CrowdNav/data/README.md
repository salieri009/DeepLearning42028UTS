# /data

Dataset root for CrowdNav preprocessing, split generation, and training input.

## Structure
- `data/raw`: source images and annotations (DVC-managed; not tracked as raw blobs in Git).
- `data/processed`: generated labels, reports, and train/val/test split outputs.

## Current Pipeline Context
- Conversion target: YOLO label files from JRDB-style annotation inputs.
- Split output: `data/processed/splits/data.yaml` generated by `split_by_sequence.py`.
- Validation report: `preprocessing_report.json` generated by `automate_preprocessing.py`.

## Dataset Sources
- JRDB (primary for current conversion flow).
- Additional datasets (COCO/Open Images/ShanghaiTech) are optional expansion sets and should be documented per experiment if used.

## Handling Rules
- Pull/push large files through DVC.
- Keep deterministic directory naming for reproducible scripts.
- Record any manual relabeling or filtering decisions in commit notes.

## Review Request Guide
- Include dataset roots used (`raw` and `processed`).
- Include whether DVC sync happened in your run.
- Include per-split counts and class mapping summary.
- Include report file path if validation was executed.
