# Data layout and DVC

- **Dataset root:** `<repository-root>/data/` (not under `train/`). Raw frames, processed labels, and YOLO splits live here; paths match `.gitignore` (large binaries via DVC when configured).
- **Training defaults:** CLI tools under `train/` resolve `data/...` against the repository root (see `train/src/repo_paths.py`), so you can `cd train` and run `python scripts/train_yolo.py` without passing `--data-yaml` when splits are at `../data/processed/splits/` from the repo perspective — i.e. `data/processed/splits/data.yaml` at repo root.
- **`data.yaml`:** Keep `path: .` relative to the directory that contains `train/`, `val/`, and `test/` (usually `data/processed/splits/`), as generated by the split step.
- **DVC:** Configuration lives in `<repository-root>/.dvc/`. Use `dvc pull` / `dvc push` from the repository root.
